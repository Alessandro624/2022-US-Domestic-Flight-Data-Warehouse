# Data Warehouse

## Overview

This directory contains the final dimensional output of the project. The files represent the star schema used for analytical queries and reporting.

Dimension tables use the `DIM_` prefix, while the central fact table stores flight-level measures and foreign keys to the dimensions.

## Expected Directory Structure

```text
data/
└── 4-data warehouse/
    ├── DIM_AIRCRAFTS.csv
    ├── DIM_CARRIERS.csv
    ├── DIM_DEP_HOURS.csv
    ├── DIM_FL_DATES.csv
    ├── DIM_JUNK.csv
    ├── DIM_STATIONS.csv
    └── FLIGHTS.csv
```

## Generation

This layer is generated by [9-ETL Pipeline.ipynb](../../phase%202%20-%20etl/9-ETL%20Pipeline.ipynb), using the dimensional design documented in [phase 1 - design](../../phase%201%20-%20design/README.md).
