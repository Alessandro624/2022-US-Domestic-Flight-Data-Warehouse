# Phase 3 - Data Visualization

This phase contains the Tableau deliverables built on top of the final data warehouse.

The visualization work translates the dimensional model into two dashboard views that support the final analytical story: where and when flight disruptions concentrate, and which operational or weather contexts help explain them.

## Dashboard Structure

| Dashboard | Analytical Role |
| --- | --- |
| When and where | Identifies the main temporal and geographical concentrations of departure disruptions. It focuses on months, weekdays, departure hours, time bands, origin airports, and airline brands. |
| Why | Explores contextual factors associated with disruption pressure, including weather events, wind gusts, visibility, temperature buckets, taxi-out time, aircraft groups, and cancellation reasons. |

## Design Notes

- Pie charts are intentionally avoided.
- Delay and cancellation rates are used instead of raw counts when comparing groups with different traffic volumes.
- High-volume filters are applied where rankings would otherwise be dominated by small airports or rare categories.
- Operational averages such as taxi-out time are computed on departed flights when cancelled flights would introduce artificial zero values.

## Data Source

The dashboards are based on the dimensional CSV files generated by the ETL pipeline in [data/4-data warehouse](../data/4-data%20warehouse/).

The workbook should remain aligned with the final dimensional design documented in [phase 1 - design](../phase%201%20-%20design/README.md), especially dimension names, time bands, measures, and analytical hierarchies.
